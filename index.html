🤖 Transformer Lektionen

🔄 RNNs vs. Transformers

RNNs: Token für Token (sequenziell) → langsam, aber einfach

Transformers: Alle Tokens parallel → schnell, stärker, moderner!

❓ Warum Transformers schneller?
✅ Parallelverarbeitung: alle Tokens gleichzeitig, nicht eins nach dem anderen

🧠 Basics: Embedding

Jedes Wort → Vector (512 dims). Embedding = Zahlenrepräsentation von Text.

Position: Wo steht das Token im Satz?

Embedding: Was ist die Bedeutung?

❓ Was macht Layer Normalization?
✅ Stabilisiert Werte → Besseres Training, weniger Probleme mit Gradienten

👁️ Self-Attention

Query: "Was suche ich?"

Key: "Bin ich relevant?"

Value: "Hier ist meine Information"

Attention(Q,K,V) = softmax(QK^T / √d) × V

Result: Jedes Token weiß, welche anderen Tokens wichtig sind!

❓ Warum √d im Nenner?
✅ Stabilisiert Gradienten & verhindert zu extreme Werte

🔀 Multi-Head Attention

Statt 1 Attention-Head: 8 verschiedene!

Head 1: Grammatik

Head 2: Semantik

Head 3: Entitäten

...und so weiter. Jeder Head spezialisiert sich!

❓ Warum 8 Heads besser als 1?
✅ Redundanz + Spezialisierung. Modell lernt mehr Pattern gleichzeitig

📍 Positional Encoding

Transformer kennt die Reihenfolge nicht von Natur aus!

Lösung: Sin/Cos Wellen pro Position

Position 0: [sin(0), cos(0), sin(0/100), cos(0/100), ...]

Position 1: [sin(1), cos(1), sin(1/100), cos(1/100), ...]

❓ Ohne Position Encoding?
✅ Transformer würde nicht verstehen: "Hund sah Katze" = "Katze sah Hund"

⚙️ Ein Block: Attention + FFN

1. Multi-Head Attention (Tokens kommunizieren)

2. Add & Norm (Residual Connection)

3. Feed-Forward Network (ReLU, 2 Layers)

4. Add & Norm (nochmal Residual)

Repeat: 6x = 6 Encoder Blocks

❓ Wozu Residual Connections?
✅ Gradienten fließen leichter. Verhindert "Vanishing Gradient Problem" in tiefen Netzen

💻 PyTorch Code

Essenz der Multi-Head Attention:

class MultiHeadAttention(nn.Module):
    def forward(self, Q, K, V):
        scores = Q @ K.T / sqrt(d_k)
        weights = softmax(scores, dim=-1)
        return weights @ V
